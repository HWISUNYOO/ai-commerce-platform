# 3. 도메인은 AI 커머스 플랫폼(일반 상품 커머스 MSA)으로 한다

- 상태: 채택(Accepted) — 2026-06-29 개정. 기존 "결제·정산 MSA" 결정을 대체.
- 날짜: 2026-06-28 (개정: 2026-06-29)

> **개정 노트:** 최초 결정은 "결제·정산 MSA"였다. 이를 **일반 상품 커머스 + AI**로
> 전환한다. 결제 도메인 전문성은 버리지 않고 `payment-service` 하나로 흡수한다.

## 배경

이직 포트폴리오로서 (a) 이직 시장 빈출 스택(MSA/PostgreSQL/Redis/Kafka/k8s/AI)을 폭넓게
보여주고, (b) 면접관에게 친숙한 도메인이며, (c) AI(RAG/Agent)를 자연스럽게 녹일 수 있는
도메인이 필요했다. 결제·정산 단독은 도메인 전문성은 깊지만 범위가 좁고 친숙도가 낮았다.

## 결정

**AI 커머스 플랫폼**을 만든다. 일반 상품 커머스를 MSA로 구성하고, AI 기능을 한 서비스로 둔다.

### 서비스 분해

| 서비스 | 책임 |
|---|---|
| `member-service` | 회원 가입/인증/프로필 |
| `product-service` | 상품 카탈로그/재고/검색 |
| `order-service` | 장바구니/주문 |
| `payment-service` | 결제 (오너의 결제·정산 도메인 전문성 활용) |
| `delivery-service` | 배송 추적/상태 |
| `ai-service` | 상품추천 RAG / 쇼핑 어시스턴트 Agent |

- 데이터: 서비스별 PostgreSQL (Database per Service)
- 통신: 동기 REST(API Gateway 경유), 비동기 Kafka 이벤트
- 캐시/멱등성: Redis
- 프론트엔드: React

## 근거

1. 모던 백엔드 스택 풀세트를 자연스럽게 데모 (억지로 끼운 기술 없음).
2. `payment-service`에서 멱등성·정합성·이벤트 처리 등 오너의 강점을 계속 드러낸다.
3. AI를 커머스 가치(추천/검색/어시스턴트)로 연결해 "AI 실무 적용"을 구체화한다.

## 결과

- 도메인 친숙도 + 모던 스택 + AI를 한 시스템에서 동시 증명.
- 6개 서비스 + 게이트웨이 + 프론트로 범위가 넓다 — 단계적 로드맵(6 Phase)으로 관리한다.
