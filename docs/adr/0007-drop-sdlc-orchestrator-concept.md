# 7. SDLC 오케스트레이터 콘셉트를 폐기한다

- 상태: 채택(Accepted) — ADR-0002의 오케스트레이터 부분을 폐기
- 날짜: 2026-06-29

## 배경

초기 콘셉트(ADR-0002)는 `PM → 분석 → 설계 → 개발 → 테스트 → 배포 → 장애분석` SDLC
전 단계를 LLM으로 오케스트레이션하는 메타 Agent를 만들고, 그 Agent가 레퍼런스
시스템을 구축하는 것이었다.

검토 결과:

1. 메타 Agent는 야심차지만 **평가·검증이 까다로운 niche**다. 누구나
   빠르게 이해하고 검증할 수 있는 결과물이 더 가치 있다.
2. 사용자가 정한 6단계 로드맵은 **표준적이고 검증 가능한 커머스 MSA**에 집중한다 —
   메타 오케스트레이터는 그 로드맵 어디에도 등장하지 않는다.
3. AI 역량은 메타 도구가 아니라 **커머스 AI 기능**(추천 RAG, 쇼핑 어시스턴트)으로
   증명하는 편이 도메인과 자연스럽게 연결된다.

## 결정

SDLC 오케스트레이터 콘셉트를 **폐기**한다. 대신 일반 상품 커머스 MSA(ADR-0003)에
집중하고, 기존에 만든 Spring Boot 오케스트레이터 코드는 **`ai-service`로 재활용**한다.

- 패키지 `dev.payflow.orchestrator` → `com.aicommerce.ai`로 이전.
- LLM 연동(LlmGateway 추상화, 구독 기반 CLI 기본)은 ADR-0005 그대로 유지하여 ai-service의
  AI 기능 구현에 사용한다.
- 그동안 검증한 자산은 보존된다: Spring Boot 3 + Java 21 스택(ADR-0002), Claude 연동
  PoC, claude.exe headless 호출 방식.

## 결과

- 결과물이 "AI 커머스 플랫폼" 하나로 단순·명확해져 이해·설명이 쉬워진다.
- 그동안의 작업(스택 셋업, LLM 연동)은 ai-service로 흡수되어 버려지지 않는다.
