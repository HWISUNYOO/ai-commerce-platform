# 1. 아키텍처 결정 기록(ADR)을 남긴다

- 상태: 채택(Accepted)
- 날짜: 2026-06-28

## 배경

이 프로젝트는 여러 마이크로서비스로 구성된 AI 커머스 플랫폼을 만든다.
의사결정의 "왜"가 코드만큼 중요하며, 설계 사고 과정을 명시적으로 기록하는 것이 핵심 가치다.

## 결정

모든 중요한 아키텍처/기술 선택은 ADR(Architecture Decision Record)로 남긴다.
형식은 Michael Nygard 스타일(배경 / 결정 / 결과)을 따른다. 번호는 순차 증가하며,
한 번 채택된 ADR은 수정하지 않고 새 ADR로 대체(supersede)한다.

## 결과

- "이 선택을 왜 했는가"에 언제든 문서로 답할 수 있다.
- 결정의 맥락이 시간이 지나도 보존된다.
- 트레이드오프를 명시적으로 기록하는 습관이 강제된다.
